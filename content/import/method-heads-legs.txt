
ТЗ: методическая раскладка темы «Головы и ноги» для цифровой платформы
Финальная методическая версия: исправлена логика задач, удалена задача 1.1, возвращены архитектурные требования к платформе.
Методическая логика темы
Главная цель темы — научить ребёнка видеть за разными сюжетами одну и ту же математическую структуру: есть несколько видов объектов, у каждого вида разный вклад в общий признак, а по суммарным данным нужно восстановить количество объектов каждого вида.
На первом уровне ребёнок решает задачи методом пробного предположения: «представим, что все были одного вида».
На втором уровне ребёнок переносит тот же способ на деньги, конфеты, очки, крылья, зубы и другие признаки.
На третьем уровне ребёнок учится проверять полноту условия: хватает ли данных для единственного ответа.
На четвёртом уровне ребёнок работает с положительными целыми ограничениями: если в задаче участвовали оба типа объектов, количество каждого типа не может быть равно нулю.
На пятом уровне ребёнок пишет решение словами, а не только вводит числовой ответ.
Ключевые методические правила
Ребёнок никогда не вставляет в условие недостающее число сам. Если число не дано, он должен сделать вывод о неполноте условия или решить задачу через натуральный перебор, если это возможно.
Если в условии сказано «мальчики и девочки», «собаки и кошки», «третьеклассники и пятиклассники», платформа ставит ограничение: каждого типа должно быть не меньше одного, если иное явно не указано.
Пример с 0 участниками допустим только тогда, когда текст задачи не утверждает участие обоих типов, например: «некоторые были одного вида, остальные — другого» и допускается, что один вид может отсутствовать.
Диагностический вывод «данных не хватает» должен быть содержательным: ребёнок объясняет, какое именно число отсутствует и почему без него нет единственного ответа.
Для каждой задачи хранится индивидуальное текстовое решение, а не один общий шаблон на всю тему.
Уровни самостоятельности
Уровень 1. Сборка модели из карточек
Ребёнок выбирает карточки объектов, перетаскивает известные числа в поля модели и собирает решение из готовых фраз. Подходит для первых задач паттерна.
Уровень 2. Дополнение пропусков
Ребёнок видит готовую структуру решения, но сам вставляет числа, действия, названия объектов и итоговый ответ в блоки [ ].
Уровень 3. Частично самостоятельное решение
Ребёнок сам выбирает, от какого предположения идти, а платформа даёт только опорные строки: «Представим...», «Тогда...», «По условию...», «Одна замена...». 
Уровень 4. Самостоятельная запись решения
Ребёнок пишет решение целиком. Платформа проверяет не дословный текст, а наличие смысловых шагов: модель, вычисление, сравнение, шаг замены, ответ.
Уровень 5. Методическая диагностика
Ребёнок определяет, что задача неполная или имеет несколько вариантов. Важно: в задачах, где оба типа участников названы как участвующие, варианты с нулём не принимаются.
Уровень 6. Натуральный перебор
Ребёнок перебирает положительные целые значения, проверяет делимость и показывает, почему остаётся один ответ или несколько возможных ответов.
Требования к экрану «Запиши решение словами»
Этот экран обязателен после каждой задачи. Его задача — сформировать навык письменного математического объяснения. Числовой ответ без текстового решения не считается завершённым прохождением задачи.
Экран открывается только после того, как ребёнок получил числовой ответ или сделал диагностический вывод.
Решение выводится строками. Каждая строка — отдельная смысловая единица: предположение, вычисление, сравнение, шаг замены, нахождение первого вида, нахождение второго вида, ответ.
Все вставляемые ребёнком элементы выделяются квадратными скобками: [число], [объект], [выражение], [вывод].
Для режима «сборка из карточек» ребёнок получает набор фраз и числовых карточек, затем располагает их в правильном порядке.
Для режима «дополнение пропусков» порядок строк уже задан, но значения внутри [ ] скрыты.
Для режима «самостоятельная запись» поле ввода пустое; рядом можно показывать чек-лист: «Есть ли предположение? Есть ли вычисление? Есть ли ответ?»
Для режима «натуральный перебор» экран должен позволять добавить несколько проверочных строк: «если ... то ...», а затем выбрать корректный вариант.
Для диагностических задач экран должен принимать вывод «единственный ответ найти нельзя», но только с объяснением причины.
Данные для разработчика
Для каждой задачи в базе должны храниться следующие поля:
task_id — номер задачи в методической структуре.
difficulty_level — уровень сложности.
pattern_id — математический паттерн.
condition_text — текст условия, который видит ребёнок.
entities — список типов объектов: например, [птицы], [звери].
features — признаки и вклад каждого объекта: ноги, колёса, сосиски, баллы, горбы и т.д.
totals — данные из условия: общее число объектов, общий признак, дополнительные ограничения.
constraints — ограничения: min_count, integer_only, both_types_present, allow_zero.
solution_mode — A/B/C/D/E: карточки, пропуски, самостоятельная запись, диагностика, натуральный перебор.
solution_lines — индивидуальный массив строк решения с блоками [ ].
accepted_answers — числовые ответы или массив допустимых вариантов.
validation_rules — правила проверки: точное число, набор вариантов, текстовый вывод, наличие ключевых слов.
hint_levels — подсказки 1–3 уровня: от мягкой наводки до почти готового шага.
Пошаговые действия ребёнка на платформе
Прочитай условие задачи.
Выбери карточки объектов, которые участвуют в задаче.
Выдели числа в условии и перетащи их в поля модели.
Проверь, хватает ли данных для единственного ответа.
Если данных хватает, выбери способ: «представим, что все были первого вида» или «представим, что все были второго вида».
Составь пробный пример.
Сравни пробное число с числом из условия.
Найди разницу.
Определи шаг замены: насколько меняется общий признак при замене одного объекта другим.
Раздели разницу на шаг замены.
Найди количество второго вида и затем первого вида.
Если задача не решается прямой заменой, выполни натуральный перебор положительных целых вариантов.
Запиши полный текстовый ответ на экране «Запиши решение словами».
Проверь, что ответ записан полным предложением и соответствует вопросу задачи.
Режимы записи решения
Режим A. Сборка из карточек
Ребёнок получает готовые карточки-фразы и карточки-числа. Он не печатает решение целиком, а собирает строки решения в правильном порядке и вставляет нужные блоки в места [ ].
Режим B. Дополнение пропусков
Ребёнок видит готовый каркас решения, но сам вставляет числа, объекты, действия и итоговый ответ в [ ]. Используется для закрепления.
Режим C. Самостоятельная запись
Ребёнок пишет решение полностью самостоятельно. Платформа проверяет наличие ключевых шагов, а не дословное совпадение с эталоном.
Режим D. Диагностический вывод
Используется для неполных задач или задач с несколькими ответами. Важно: если текст утверждает участие обоих типов объектов, варианты с 0 не принимаются.
Режим E. Натуральный перебор / положительные ограничения
Используется, когда прямого общего числа объектов или общего признака недостаточно для стандартной замены, но можно перебрать положительные целые варианты. Все варианты с нулём запрещены, если оба типа названы как участвующие.
Сложность 1. Базовая замена: два вида объектов и один признак
Паттерн 1. Головы / ноги / колеса / конечности
1.1. Звери и птицы
Условие: В зоопарке живут четвероногие звери и двуногие птицы. В зоопарке имеется 30 голов и 100 ног. Сколько зверей и сколько птиц живет в зоопарке?
Сбор модели условия на платформе: Ребенок собирает модель из карточек: [два вида объектов], [общее количество], [общий признак], [признак первого вида], [признак второго вида]. Если какого-то общего числа нет в условии, ребенок не придумывает его, а отмечает «данных не хватает».
Шаг «Запиши решение словами»: Режим A: ребенок собирает решение из карточек-строк и карточек-чисел.
Текстовое решение с блоками для вставки:
Представим, что все [30] животных были [птицами].
Тогда ног было бы [30 x 2 = 60].
По условию ног [100].
Не хватает [100 - 60 = 40] ног.
Один зверь добавляет [4 - 2 = 2] ноги.
Значит, зверей было [40 ÷ 2 = 20].
Птиц было [30 - 20 = 10].
Ответ: [20 зверей и 10 птиц].
1.2. Цыплята и змеи
Условие: В инкубаторе лежало 120 яиц. Из некоторых вылупились цыплята, а из остальных — змеи. В сумме у детёнышей оказалось 162 ноги. Сколько вылупилось змей?
Сбор модели условия на платформе: Ребенок собирает модель из карточек: [два вида объектов], [общее количество], [общий признак], [признак первого вида], [признак второго вида]. Если какого-то общего числа нет в условии, ребенок не придумывает его, а отмечает «данных не хватает».
Шаг «Запиши решение словами»: Режим A: ребенок собирает решение из карточек; числовые блоки [120], [0], [162], [2], [81], [39] вставляет сам.
Текстовое решение с блоками для вставки:
Представим, что все [120] детёнышей были [змеями].
Тогда ног было бы [0].
По условию ног [162].
Один цыплёнок добавляет [2] ноги.
Поэтому цыплят было [162 ÷ 2 = 81].
Значит, змей было [120 - 81 = 39].
Ответ: [39 змей].
1.3. Гусята и крокодильчики
Условие: В инкубаторе лежало 200 яиц. Из некоторых вылупились гусята, а из остальных — крокодильчики. Всего из яиц вышло 500 ног. Сколько вылупилось гусят и сколько крокодильчиков?
Сбор модели условия на платформе: Ребенок собирает модель из карточек: [два вида объектов], [общее количество], [общий признак], [признак первого вида], [признак второго вида]. Если какого-то общего числа нет в условии, ребенок не придумывает его, а отмечает «данных не хватает».
Шаг «Запиши решение словами»: Режим A/B: первая строка собирается из карточек, остальные строки дополняются числами.
Текстовое решение с блоками для вставки:
Представим, что все [200] детёнышей были [гусятами].
Тогда ног было бы [200 x 2 = 400].
По условию ног [500].
Лишних ног получилось [500 - 400 = 100].
Один крокодильчик добавляет [4 - 2 = 2] ноги.
Крокодильчиков было [100 ÷ 2 = 50].
Гусят было [200 - 50 = 150].
Ответ: [150 гусят и 50 крокодильчиков].

Сбор модели условия на платформе: Ребенок сам переносит данные из условия в поля модели: [всего объектов], [общий признак], [признак 1], [признак 2]. Карточки используются только как подсказки названий полей.
Шаг «Запиши решение словами»: Режим B: ребенок дополняет пропуски в готовом тексте решения.
Текстовое решение с блоками для вставки:
Представим, что все [40] существ были [жуками].
Тогда ног было бы [40 x 6 = 240].
По условию ног [270].
Лишних ног [270 - 240 = 30].
Один паук добавляет [8 - 6 = 2] ноги.
Пауков было [30 ÷ 2 = 15].
Жуков было [40 - 15 = 25].
Ответ: [25 жуков и 15 пауков].
1.5. Жуки и пауки: правые ноги
Условие: В банке сидят жуки и пауки, всего у них 70 туловищ и 242 правые ноги. Сколько в банке жуков и сколько пауков?
Сбор модели условия на платформе: Ребенок сам переносит данные из условия в поля модели: [всего объектов], [общий признак], [признак 1], [признак 2]. Карточки используются только как подсказки названий полей.
Шаг «Запиши решение словами»: Режим B: ребенок дополняет готовый каркас, но сначала сам выводит число правых ног у жука и паука.
Текстовое решение с блоками для вставки:
У жука [6] ног, значит правых ног [3].
У паука [8] ног, значит правых ног [4].
Представим, что все [70] существ были [жуками].
Тогда правых ног было бы [70 x 3 = 210].
По условию правых ног [242].
Лишних правых ног [242 - 210 = 32].
Один паук добавляет [4 - 3 = 1] правую ногу.
Пауков было [32 ÷ 1 = 32].
Жуков было [70 - 32 = 38].
Ответ: [38 жуков и 32 паука].
1.6. Велосипедисты
Условие: Вася увидел 35 велосипедистов. Некоторые катались на двухколесном велосипеде, а другие на трехколесном. Всего Вася насчитал 102 колеса. Сколько всего двухколесных велосипедов увидел Вася?
Сбор модели условия на платформе: Ребенок сам переносит данные из условия в поля модели: [всего объектов], [общий признак], [признак 1], [признак 2]. Карточки используются только как подсказки названий полей.
Шаг «Запиши решение словами»: Режим B: ребенок дополняет пропуски и отдельно отмечает, что задача спрашивает не трехколесные, а двухколесные велосипеды.
Текстовое решение с блоками для вставки:
Представим, что все [35] велосипедистов ехали на [двухколёсных] велосипедах.
Тогда колёс было бы [35 x 2 = 70].
По условию колёс [102].
Лишних колёс [102 - 70 = 32].
Один трёхколёсный велосипед добавляет [3 - 2 = 1] колесо.
Трёхколёсных велосипедов было [32].
Двухколёсных велосипедов было [35 - 32 = 3].
Ответ: [3 двухколёсных велосипеда].

Условие: На парковке магазина было припарковано 42 двухколесных и трехколесных велосипеда. Всего колес было 88. Сколько трехколесных велосипедов было припарковано на парковке?

Шаг «Запиши решение словами»: Режим B: ребенок дополняет пропуски в тексте решения.

Представим, что все [42] велосипеда были [двухколёсными].

1.8. Мухи и слоны
Условие: Мухи и слоны играют в футбол. Всего 10 игроков. По полю топает 48 ног. Запишите счет матча МУХИ:СЛОНЫ.
Сбор модели условия на платформе: Ребенок сам переносит данные из условия в поля модели: [всего объектов], [общий признак], [признак 1], [признак 2]. Карточки используются только как подсказки названий полей.
Шаг «Запиши решение словами»: Режим B: ребенок дополняет решение и в конце записывает ответ в специальном формате [МУХИ:СЛОНЫ].

Представим, что все [10] игроков были [слонами].

1.9. Дроиды и генерал Гривус
Условие: У дроида 2 ноги и 2 руки, а у Генерала Гривуса 2 ноги и 4 руки. Всего видно 20 ног и 26 рук. Сколько там дроидов?
Сбор модели условия на платформе: Ребенок самостоятельно выделяет данные из условия и записывает модель без готовой таблицы; платформа проверяет ключевые числа и смысловые шаги.
Шаг «Запиши решение словами»: Режим C: ребенок пишет решение сам, но платформа требует два этапа: сначала найти общее число существ по ногам, потом решить по рукам.
Текстовое решение с блоками для вставки:
У каждого существа по [2] ноги, поэтому всего существ [20 ÷ 2 = 10].
Представим, что все [10] существ были [дроидами].
Тогда рук было бы [10 x 2 = 20].
По условию рук [26].
Лишних рук [26 - 20 = 6].
Один генерал Гривус добавляет [4 - 2 = 2] руки.
Генералов Гривусов было [6 ÷ 2 = 3].
Дроидов было [10 - 3 = 7].
Ответ: [7 дроидов].

Не использовать один общий шаблон для всех задач. Для каждой задачи хранить индивидуальный массив строк решения.
Каждый блок в квадратных скобках [ ] хранить как отдельное интерактивное поле: number, object, operation, expression, conclusion.
Для режима A показывать карточки строк и карточки вставок; проверять порядок строк и корректность вставок.
Для режима B показывать строки в правильном порядке, но скрывать значения внутри [ ].
Для режима C показывать только инструкцию «Запиши решение словами»; проверять ключевые элементы решения, а не дословное совпадение.
Для режима D принимать диагностический вывод только с объяснением причины. Если оба типа объектов названы как участвующие, варианты с 0 запрещены.
Для режима E поддержать таблицу или карточки натурального перебора: значение первого вида, остаток, проверка делимости, вывод.
Если задача неполная, ребёнок не должен сам добавлять недостающее число. Добавлять данные может только методист/редактор в тексте задачи.
В конце каждого решения ребёнок обязательно записывает строку «Ответ: ...» или «Вывод: ...».
Проверка текстового решения должна быть смысловой: допускаются разные формулировки при сохранении математических шагов.
Для задач с положительными ограничениями хранить параметры min_count: 1 и both_types_present: true.
