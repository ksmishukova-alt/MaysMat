
Аудит скрытых и архивных задач: перевод в детскую траекторию
Проверка проведена по актуальному архиву проекта MaysMat и spot-check production-страниц. Цель: определить, какие скрытые/архивные/методистские задачи можно переводить в детский маршрут, какие требуют исправлений, и какие должны остаться в paper-mode или олимпиадном архиве.
1. Итог тестирования
Всего задач в экспортированной базе: 272.
Проверено скрытых/архивных/методистских задач: 235.
Распределение по publishTier: methodistOnly: 165, hidden: 7, archive: 63
Распределение по qaStatus: ready: 141, needsReview: 87, blocked: 7
Ключевой вывод: переводить весь скрытый/архивный банк в детскую траекторию одним релизом нельзя. Нужно выпускать волнами: сначала готовые methodistOnly без issues, затем задачи с внешними ссылками после раскрытия решения, затем визуальные задачи после SVG/схем, а архивные конструкции — только через paper-mode и проверку взрослым.
2. Проверенные технические факты
qa:task-access: guard доступа согласован с publishing; ребёнок не должен видеть hidden/archive/methodistOnly в обычном режиме.
qa:task-quality: детский маршрут без блокирующих issues; во всём банке остаются P1/P0-задачи.
qa:categories: общие “кролики/клетки” не обнаружены, но многим темам нужны специализированные экраны.
qa:smoke: constructions по умолчанию показывает 0 задач и раскрывает 63 задачи только через режим архива.
Production spot-check: главная и ветки dirichlet / unlucky / constructions / cutting открываются; детские списки отфильтрованы.
3. Рекомендуемая стратегия перевода
Волна 1.5: Расширить базовый Дирихле: methodistOnly + ready + no issues. Добавить routeOrder, экран видео/объяснения и progressionProfile.
Волна 2: Метод неудачника, продвинутые Головы-ноги и Сказочные пещеры после answerKey/шаблонов.
Волна 3: Остатки и геометрическое разбиение. Обязательны отдельные видео и специализированные экраны.
Волна 4: Табличные значения, связи/графы, раскраски. Не использовать универсальный Дирихле-flow.
Волна 5: Конструкции/контрпримеры и принцип крайнего: только как отдельные темы, преимущественно paper-mode.

6. Проверка по ролям
10-летний ребёнок: В обычном режиме скрытые и архивные задачи не видны. Если открыть их в режиме методиста/архива, ребёнок чаще всего застревает на отсутствии визуала, внешней ссылке “см. задачу”, слишком общем flow или необходимости доказательства на бумаге.
Методолог: Готовые methodistOnly-задачи можно повышать волнами, но каждый блок требует отдельного видео-ввода и снятия опоры: 1–3 задачи объясняют метод, 4–10 тренируют, 11–15 требуют выбора действия, 16+ уходят в запись выражений/paper-mode.
UX/UI senior: Нужны разные экраны для разных типов мышления: остатки-домики, геометрическое разбиение, граф связей, таблица значений, paper-mode. Универсальный экран “зайцы/клетки” недостаточен.
Архитектор: Guard и publishTier работают. Следующий слой — progressionProfile, visualAsset, paperBlankTemplate, parentReviewPolicy, taskReadinessChecklist.
Системный аналитик: Для перевода в детский маршрут задача должна иметь: method, screenFlow, answerKey, solutionText, hints, routeOrder, supportMode, validation, visualStatus, paperPolicy.
Разработчик: Нужны компоненты: RemainderCellsStep, GeometryPartitionStep, GraphRelationsStep, ValuesRangeStep, ConstructionPaperStep, BlankDownloadButton, ParentReviewGate.

8. Definition of Done для перевода задачи в childRoute
Задача имеет publishTier=childRoute или training и qaStatus=ready.
Нет issues: contains_N_equals_N, contains_external_reference, missing_visual_asset, missing_answer_key, incomplete_condition.
Есть полноценный answerKey и эталонный текст решения без “см. задачу”.
Есть специализированный screenFlow под тему, а не только общий Дирихле-flow.
Есть progressionProfile: объяснение → тренировка → выбор действия → вписывание выражений → бумажное решение.
Если задача геометрическая/раскрасочная/графовая, есть визуальный asset или интерактивная схема.
Если задача уровня 16+ или archive, есть бумажный режим: скачать бланк, загрузить решение, parentId/submissionId, pending/approved/redo.
Задача проверена в роли ребёнка: можно пройти без методиста и без внешнего опорного конспекта.
