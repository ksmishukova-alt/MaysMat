# Guided-lite MVP: архитектура task-runner и соответствие банку

> Версия: 1.0 · Июнь 2025  
> Источник docx: `Полная_архитектура_task_runner_Головы_и_Ноги_все_задачи.docx`  
> **Эталон ID задач — банк платформы** (`catalog.generated.ts`, 51 задача)

---

## 1. Решение по расхождениям с docx

| Вопрос | Решение |
|--------|---------|
| FactExtractor (полная модель из 5 карточек) | **Guided-lite**: `table_input` + worksheet «Собери данные» — целевой MVP |
| Отдельный экран «Числовой ответ» | **Не делаем** — проверка в worksheets + `word_solution` |
| 52 сценария в docx vs 51 в банке | **Банк не меняем**; docx перенумеровать под банк |
| «3.3 Мороженое» в docx | **В резерв** — не в банке; см. §4 |

---

## 2. Карта экранов docx → реализация (guided-lite)

| № | Docx | Компонент | Реализация |
|---|------|-----------|------------|
| 0 | Вход | TaskIntro | Карточка задачи |
| 1 | Чтение | ConditionReader | `ReadConditionStep` + **подсветка чисел и ключевых слов** |
| 2 | Участники | EntityPicker | `drag_select` (custom для 6.x, 7.4, 7.6, 6.5) |
| 3 | Модель | FactExtractor | `table_input` + `-totals` worksheet |
| 4 | Полнота | SufficiencyCheck | Custom: **3.1**, **3.7**, **7.6**; иначе в worksheets |
| 5 | Метод | ModelBuilder | `single_select` «Представим, что все…» |
| 6 | Расчёт | StepSolver | `worksheet_table` или **custom middle** |
| 7 | Число | AnswerEntry | *(пропущен — by design)* |
| 8 | Текст | WrittenSolution | `word_solution` (режимы A–E) |
| 9 | Итог | ValidationSummary | `auto_explanation` preview |

**Фазы UI** (`screen-architecture.ts`): 3–5 блоков, не 10 отдельных экранов.
