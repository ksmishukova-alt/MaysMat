Аудит по ролямРебёнок 10 летНа задачах, где экран показывает N = N или M = M, ребёнок не выполняет действие, а видит незаполненную формулу. Это ломает навык пошаговой записи.В «Методе неудачника» ребёнок получает неправильные сущности: например, монетная задача превращается в задачу про шарики. Это создаёт недоверие к платформе.Геометрические задачи без рисунка перегружают воображение: ребёнок должен сначала увидеть разбиение, а уже потом делать вывод Дирихле.МетодологТема после пересборки стала методически чище, но в продукте надо разделить: базовый Дирихле, неудачник, остатки, геометрия, таблицы, графы, advanced-архив.Задачи L4/L5 нельзя показывать в одной линейке с L1: они требуют другого формата проверки — свободное доказательство, рецензирование, эталон после попытки.Много задач с подзадачами а/б нужно дробить, иначе прогресс и проверка ответа становятся нечёткими.UX/UI seniorНужны разные screen templates: direct pigeonhole, generalized, unlucky, remainder, geometry partition, table values, graph relations, construction/counterexample.Для «свободного доказательства» L5 нельзя использовать тот же UI, что для L1–L2. Нужен режим: план → черновик → самопроверка → эталон.Задачи с visual_required должны иметь отдельный «экран рисунка/построения»; простого текста недостаточно.АрхитекторСейчас генератор пытается вывести зайцев/клетки автоматически для слишком разных типов задач. Нужны ручные model cards для каждой задачи или хотя бы для каждого подтипа.Нужен preflight validator: если в пользовательском UI остаются N/M без числа, задача не может идти в guided-режим.Необходимо отделить task-bank от task-runner: задача может быть в банке, но не иметь готового интерактивного сценария.Системный аналитикДля каждой задачи нужны поля readiness: publishable, needs_visual, needs_manual_model, answer_key_complete, suitable_for_child_path.Для multi-part задач нужны subtask objects, а не один общий condition string.Для ссылок «см. задачу» нужен процесс разворачивания в полноценный solutionKey.РазработчикДобавить QA-тест: ни один экран сравнения не должен содержать строку «N = N» или «M = M» в guided-режиме.Добавить QA-тест на сущности: correct options не должны быть обрывками слов вроде «Красны» и не должны подставлять «цвета шариков» в задачу про монеты.Добавить контентный тест: в answerKey не допускаются «См. задачу», «Максимум баллов», «Задача #…» без развёрнутого решения.5.
